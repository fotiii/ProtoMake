# Настройка отправки заявок из формы

Сайт умеет отправлять заявки сразу в Telegram и на почту через Resend.

## Какие переменные нужны в Vercel

- `TELEGRAM_BOT_TOKEN` — токен Telegram-бота.
- `TELEGRAM_CHAT_ID` — chat id, куда бот должен слать заявки.
- `RESEND_API_KEY` — API ключ Resend.
- `NOTIFY_EMAIL_FROM` — подтверждённый sender, например `PROTOMAKE <onboarding@resend.dev>` для теста или ваш домен.
- `NOTIFY_EMAIL_TO` — почта, на которую нужно получать заявки.

## Как добавить переменные в Vercel

1. Откройте проект `proto-make` в Vercel.
2. Перейдите в `Settings` → `Environment Variables`.
3. Добавьте все нужные значения.
4. Сохраните изменения.
5. После сохранения сделайте новый redeploy или новый `git push`.

## Минимальная рабочая схема

Если хотите сначала проверить только Telegram:

- задайте `TELEGRAM_BOT_TOKEN`
- задайте `TELEGRAM_CHAT_ID`

Если хотите сначала проверить только почту:

- задайте `RESEND_API_KEY`
- задайте `NOTIFY_EMAIL_FROM`
- задайте `NOTIFY_EMAIL_TO`

Форма считает отправку успешной, если хотя бы один канал сработал.

## Как получить Telegram Chat ID

Один из самых простых способов:

1. Создать бота через BotFather.
2. Написать что-нибудь боту.
3. Добавить бота в нужный чат или оставить личный диалог.
4. Открыть в браузере:

`https://api.telegram.org/bot<ВАШ_ТОКЕН>/getUpdates`

5. Найти в ответе `chat.id`.

## Что делает серверная часть

Файл `api/quote-request.js`:

- принимает заявку из формы
- валидирует основные поля
- отправляет сообщение в Telegram
- отправляет письмо через Resend
- возвращает номер заявки и список успешно отработавших каналов
